# 发布与在线更新

## 更新源格式

发行版内置 CampusMetric 官方更新源，设置页不会显示具体地址；管理员不做修改即可直接检查更新。学校如有自己的更新服务器，也可以填写自定义地址覆盖官方源，清空后恢复默认。

更新源目录包含：

- `version.json`
- `campus-evaluation-update-x.y.z.zip`

`version.json` 字段见 `release/version.example.json`。系统会校验 SHA256，解压时防止路径穿越，更新时显式排除：

- `instance/`（SQLite）
- `uploads/`（考评材料）
- `backups/`
- `.env`
- `.venv/` 和免安装包的 `env/`
- `.git/`

应用更新时除备份被替换的代码外，也会先备份 `instance/evaluation.sqlite` 到同一回滚目录；数据库兼容迁移只在新版本重新启动后执行。

## 生成更新包

```bash
python scripts/build_update_package.py 0.3.7
```

输出到 `dist/`，包含 zip 和已计算 SHA256 的 `version.json`。发布前必须先在
`evaluation_app/changelog.py` 顶部添加同版本日志；构建脚本会校验版本号，
并把对应日志自动写入更新清单，避免发布包与更新说明脱节。

## 免安装包

Windows 免安装包（`scripts/build_windows_portable.py`）根目录包含完整项目代码和预装依赖的 `env/` Python 运行时，用户双击 `CampusMetric-启动.bat` 即可运行。启动器会设置 `CE_MANAGED_LAUNCHER=1`，在线更新应用后可自动重启。

macOS 免安装包（`scripts/build_macos_portable.py`）内置 python-build-standalone 的 Apple Silicon（arm64）与 Intel（x86_64）双架构运行时，依赖 wheels 按架构分别预装；用户双击 `CampusMetric-启动.command`，启动器清除下载隔离标记后按本机芯片自动选择运行时。打包必须使用 `ditto -c -k --keepParent` 以保留符号链接和执行权限。

开源仓库不直接提供安装包下载，安装包在项目官网发布。当前构建脚本统一使用 Python 3.13.14（Windows 用官方 embeddable runtime，macOS 用 python-build-standalone），并下载与目标平台匹配的 wheels；发布前必须检查 ZIP 完整性、依赖文件和敏感文件排除结果。

源码开发模式可检查、下载、校验和暂存更新，但不会自动覆盖当前开发目录。

为确保旧版免安装包在无网络环境升级后也能读取 PDF，更新包内置纯 Python `pypdf` 运行库及其 BSD-3-Clause 许可证；正常源码安装仍以 `requirements.txt` 为准。

## 开源发布边界

公开仓库和发布包不得包含 SQLite 数据库、上传材料、AI API Key、学校正式账号或私有更新源地址。后续固化并推送 GitHub 时应使用 `buBailai` 身份。
